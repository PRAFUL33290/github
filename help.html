📚 Aide - Comment utiliser cette application

Guide complet pour tirer le meilleur parti du GitHub Terminal Command Center.

🎯 Qu'est-ce que cette application ?

Le GitHub Terminal Command Center est une page web interactive qui regroupe les commandes Git et GitHub CLI les plus utiles. Elle est conçue pour t'aider à :

  • Apprendre et mémoriser les commandes Git essentielles
  • Copier rapidement les commandes dont tu as besoin
  • Filtrer les commandes par catégorie ou par recherche
  • Avoir une référence rapide toujours accessible

🔍 Utilisation de la recherche

La barre de recherche te permet de trouver rapidement les commandes dont tu as besoin :

  • Tape un mot-clé comme "commit", "branch", "stash", "rebase"
  • La recherche fonctionne sur le titre, la commande et la description
  • Les résultats sont filtrés en temps réel au fur et à mesure de ta saisie
  • Les sections vides sont automatiquement masquées
Astuce : Tu peux également utiliser le menu déroulant des catégories pour filtrer les commandes par thème (Configuration, Branches, Merge, etc.).

📋 Copier les commandes

Chaque commande dispose d'un bouton "Copier" qui te permet de la copier instantanément dans ton presse-papiers :

  1. Clique sur le bouton "Copier" à côté de la commande
  2. Le bouton affiche "Copié" pendant 2 secondes pour confirmer
  3. Colle la commande directement dans ton terminal avec Ctrl+V (ou Cmd+V sur Mac)

Copier plusieurs commandes

Tu peux aussi copier toutes les commandes visibles d'un coup :

  • Utilise le bouton "Copier les commandes visibles" en haut de la page
  • Particulièrement utile après avoir appliqué un filtre de recherche

🏷️ Filtrage par catégorie

Les commandes sont organisées en catégories thématiques :

  • Commandes Essentielles - Les commandes de base à connaître absolument
  • Configuration Initiale - Paramètres et configuration de Git
  • Branches - Création, changement, et gestion des branches
  • Push, Pull, Fetch - Synchronisation avec GitHub
  • Merge & Rebase - Fusion et réorganisation de l'historique
  • Stash - Mise de côté temporaire des modifications
  • Undo & Reset - Annulation et retour en arrière
  • Tags & Releases - Gestion des versions
  • GitHub CLI (gh) - Commandes GitHub en ligne de commande
  • Et bien d'autres...

Sélectionne une catégorie dans le menu déroulant pour afficher uniquement les commandes de cette catégorie.

🔧 Raccourcis et astuces

Raccourcis clavier

  • Clique dans la barre de recherche et commence à taper pour filtrer
  • Utilise Tab pour naviguer entre les éléments

Bonnes pratiques

  • Lis la description de chaque commande avant de l'utiliser
  • Certaines commandes sont puissantes et peuvent être destructrices (ex: git reset --hard)
  • Utilise toujours git status avant et après chaque opération importante
  • Crée une branche temporaire avant d'effectuer des opérations risquées
  • Préfère --force-with-lease à --force pour les push
Important : Fais toujours attention aux commandes avec --force, --hard, ou clean -fd. Elles peuvent supprimer définitivement des modifications non sauvegardées.

💡 Aide supplémentaire

Commande git help

Pour obtenir de l'aide sur n'importe quelle commande Git directement dans ton terminal :

  • git help - Affiche l'aide générale de Git
  • git help <commande> - Affiche l'aide détaillée d'une commande spécifique
  • Exemple : git help commit ou git commit --help

Ressources en ligne

  • Documentation officielle Git : git-scm.com/doc
  • GitHub Docs : docs.github.com
  • GitHub CLI Docs : cli.github.com/manual
