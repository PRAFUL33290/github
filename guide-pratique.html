Le workflow Git, de A à Z

Scénario : tu ouvres ton projet dans un Codespace et tu veux travailler dessus. Voici les 7 étapes, dans l'ordre, avec le pourquoi à chaque fois. C'est toujours la même boucle.

← Retour aux commandes

Avant tout : les 3 endroits où vit ton code

  1. Tes fichiers dans le Codespace — ce que tu modifies.
  2. Le dépôt Git local — les « photos » (commits) de ton travail.
  3. GitHub (le distant, origin) — la copie en ligne, la vraie sauvegarde.

Tout le workflow consiste à faire circuler ton travail : fichiers → commit → push vers GitHub. Rien de plus.

La boucle à retenir

main propre → branche → travail → add / commit → push → Pull Request → merge → retour sur main → pull

Un doute ? git status ne casse jamais rien et te dit toujours où tu en es.
